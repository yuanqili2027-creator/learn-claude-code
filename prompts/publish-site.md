# 建站与发布（从零到上线）

> 适用：把任何文件夹/页面发布成公开网站。

```text
帮我把[这个文件夹 / 我们刚做好的页面]发布到 GitHub Pages：
1. 建一个叫 [仓库名] 的公开仓库；
2. 推送全部文件；
3. 开启 GitHub Pages；
4. 轮询验证网站真的能访问（HTTP 200），把最终网址发给我。
```

# 后续更新

```text
我改了[哪些内容/你帮我把 XX 改成 YY]，改完提交推送，
等部署生效后验证线上已更新，再告诉我完成。
```

**要点**：①「验证后再报告」防止「应该好了」式交差；②仓库名出现在网址里，起个像样的英文短名；③更新同理——推送不等于生效，要求它确认线上版本。

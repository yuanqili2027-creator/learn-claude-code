# 并行干活：subagent 与后台任务

> 适用：方案已确认、施工需要较长时间，而你还想继续和 AI 讨论别的事。

```text
方案就按刚才确认的来。你派一个 subagent 在后台去完成全部工作
（包含建仓库、写代码、部署、验证的完整任务书），
完成后向我汇报结果。我们现在继续讨论[下一件事]。
```

# 让 AI 出去做调研

```text
派几个 agent 去搜一下[主题]的实战案例、最佳实践和踩坑经验，
要求来源真实可查（附链接），过滤掉营销号和内容农场，
最后整理成一份分好类的中文笔记给我。
```

**要点**：①只有「方案已确认、边界清晰」的工作适合外派——需求还在变的不要派；②任务书越完整，返工越少：目标、约束、验收标准都要在派单时讲清；③调研类任务明确要求「附来源链接 + 过滤 SEO 垃圾」，质量差别巨大。

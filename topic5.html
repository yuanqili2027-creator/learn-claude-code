Claude Code 初识

能真正操作电脑的 AI：权限、token、模型与订阅计费

Claude Code：能直接操作你电脑的 AI 命令行工具

一、它到底是什么：会「动手干活」的 AI

我们平时用的网页版 Claude、ChatGPT，本质是一个「聊天框」——你问它答，它只会输出文字，看不到也碰不到你电脑里的任何东西。你想让它帮你改一份文件，只能自己复制粘贴进去、再把结果复制出来。

Claude Code 不一样。它是一个运行在「终端」（Terminal，就是那个黑色的命令行窗口）里的 AI 助手，最大的区别是：它能真正操作你的电脑。 按官方说法，它是一个「agentic（自主代理式）」工具——你给一句自然语言指令，它会自己规划步骤、自己调用工具、一步步把事做完，中途不需要你手把手带。

课堂上老师的演示就是最好的例子：把一份考卷丢进文件夹，一句「用 LaTeX 写出完整解答并编译成 PDF」，Claude Code 就自己去执行 ls（列目录）、读文件、检查电脑上有没有装 LaTeX 工具、写出 .tex 源文件、再运行编译命令生成 PDF——全程它自己动手。这在网页版里是绝对做不到的。

一句话总结区别：网页版 Claude 是「隔着玻璃跟你聊天」，Claude Code 是「坐到你电脑前帮你干活」。

二、它能干什么：远不止写代码

虽然名字里有「Code」，但它的能力对非程序员一样有用。官方列出的核心能力包括：读写文件、运行命令、联网查资料、连续多轮自主完成任务、管理整个项目文件夹。落到日常场景：

  • 整理资料：把一堆杂乱文件按规则批量重命名、归类到不同文件夹。
  • 读 PDF 做总结：把几十页的报告或论文丢给它，让它读完写摘要、提要点。
  • 排版作业 / 文档：像演示里那样，用 LaTeX 或 Markdown 排版，直接生成 PDF。
  • 做简历、写文案：给它素材，让它生成并反复修改成品文件。
  • 上网查证：让它联网搜最新信息，再整理成一份文档。

因为它能「看到」并管理你整个项目目录，处理跨几十个文件的大任务时，比网页版复制粘贴强太多。

三、安装与启动

前提：一台 Mac、Linux 或 Windows 电脑。以下为官方推荐的安装方式（请以官方文档为准）：

  • 原生安装脚本（官方首选）——macOS / Linux 终端里粘贴： curl -fsSL https://claude.ai/install.sh | bash （这种方式会在后台自动更新到最新版。）
  • Homebrew（Mac 用户常用）：brew install --cask claude-code
  • Windows 用户可用 PowerShell 脚本或 winget install Anthropic.ClaudeCode。

装好后，用 cd 命令进入你要处理的项目文件夹，输入一个词回车即可启动：

cd 你的项目文件夹
claude

首次使用它会提示你登录（用你的 Claude 账号授权），登录一次以后就记住了。除了终端，Claude Code 现在也有桌面 App、VS Code 插件和网页版，但终端版功能最全。

四、权限与授权机制：安全是核心

「AI 能操作我电脑」听起来有点吓人——万一它删错文件怎么办？Claude Code 的设计正是为此：做任何有风险的操作（写文件、删文件、运行命令）之前，它会停下来向你请求许可，屏幕上弹出「是否允许执行 XXX」，你确认了它才动手。

所谓「授予权限」，就是你在为这一步操作点头。安全建议：

  • 一开始逐条看清楚再批准，别无脑一路回车。
  • 对「读文件、列目录」这类安全操作可以放心;对「删除、覆盖、联网发送」多留个心眼。
  • 在重要资料的文件夹里操作前，先备份;或先在测试文件夹里熟悉它的脾气。

五、常用交互：怎么跟它对话

  • 给指令：直接用大白话打字，中文英文都行。想让它全程说中文，开头说一句「请全程用中文回答」即可。
  • 斜杠命令：输入 / 开头的命令控制程序本身。最常用的是 /help（查看所有命令）；此外还有清空对话、切换模型、查看用量等命令。
  • 中断：它跑偏了或你想喊停，按 Esc 打断当前动作；Ctrl+C 强制中止。
  • 退出：输入 /exit，或按两次 Ctrl+C。

六、token 与上下文窗口

token（词元）是 AI 眼里的「字」，是它计量和收费的基本单位。粗略换算：一个中文字大约 1 个 token，一个英文单词大约 1–2 个 token。你发的问题、它给的回答，都会被拆成一个个 token 来计数。

上下文窗口（context window），可以理解成 AI 的「短期记忆容量」——它一次对话里最多能「记住」多少 token。目前旗舰模型的窗口已达上百万 token（约等于好几本书），但也不是无限。对话拉得太长会占满窗口，这时它会自动「压缩」早期内容来腾地方。所以任务做完了开个新对话（清空上下文），既省额度也更清爽。

七、模型与订阅计费

模型：Claude 家族按能力从低到高分 Haiku（快而省）、Sonnet（均衡主力）、Opus（旗舰）几档。老师课上说的「Opus 是旗舰」基本没错——当前旗舰是 Opus 4.8；不过 2026 年官方在 Opus 之上还推出了更强的 Fable 5。日常够用的默认模型是 Sonnet 5。（模型迭代很快，具体请以官方页面为准。）

计费——两种模式别搞混：

  1. 订阅制（推荐新手）：按月付固定费用，Claude 网页版和 Claude Code 共用同一份额度。
  2. API 按量计费：按实际用掉的 token 付费，适合开发者接程序。注意：如果你电脑里设了 ANTHROPIC_API_KEY 环境变量，Claude Code 会走 API 扣费，容易「意外收到账单」——想用订阅额度，就别设这个变量。

订阅档位（价格以 官方定价页 为准）：

  • Pro，约 $20/月：适合轻度、短时使用，官方定位「小项目短时冲刺」。
  • Max，约 $100/月（5 倍用量）与 $200/月（20 倍用量）：重度使用、输出上限更高、高峰期优先。

为什么有额度限制：官方采用「滚动 5 小时窗口 + 每周额度」的双层机制。用超了不是当天封死，而是等下一个时间窗口刷新（通常几十分钟）。官方只公布倍数（Pro 1×、Max 5×/20×），不公布精确 token 数——网上任何「每窗口 X 个 token」都是第三方估算，别当准。

务实建议：先花 $20 上 Pro 用满一周正常工作量，一周内只偶尔撞到几次上限就够用；如果频繁被限流，再升 Max 5×;Max 20× 只有整天高强度、多开会话的人才需要。
